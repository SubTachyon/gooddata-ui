Specify Table Totals

There are several examples that can show how to define table totals, how they look in the UI and what has to be defined in ExecutionObject (AFM and ResultSpec) to achieve expected results.

Currently supported types

TypeIn 1st dimensionIn 2nd dimension
Grand totalsDONENOT YET
Sub-totalsNOT YETNOT YET

Grand totals

Grand totals are totals that are computed for the measure in a whole column or row.

In 1st dimension

  • a1, a2 are attributes.
  • m1 is a measure.

Example: ExecutionObject for grand totals in 1st dimension

execution : {
    afm: {
        ...
        nativeTotals: [
            {
                measureIdentifier: 'm1',
                attributeIdentifiers: []
            }
        ]
    },
    resultSpec: {
        ...
        dimensions: [
            {
                itemIdentifiers: ['a1', 'a2'],
                totals: [
                    {
                        measureIdentifier: 'm1',
                        type: 'sum',
                        attributeIdentifier: 'a1'
                    },
                    {
                        measureIdentifier: 'm1',
                        type: 'nat',
                        attributeIdentifier: 'a1'
                    }
                ]
            },
            {
                itemIdentifiers: ['measureGroup'],
                totals: []
            }
        ]
    }
}
